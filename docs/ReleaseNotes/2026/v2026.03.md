# Release Notes - v2026.03

## Overview

This release introduces major enhancements for remote workload hosting, comprehensive job scheduling capabilities, and improved operational APIs. The update focuses on production-ready remote hosting features, making it easier for workload developers to deploy and manage Fabric workloads at scale.

## 🚀 New Features

### Remote Workload Hosting (Production Ready)

#### Public Setup Scripts
- **SwitchToRemoteHosting.ps1**: New script enabling remote hosting deployment to production environments
- **CreateJob.ps1**: Script for enabling job scheduling capabilities on workload items

#### Remote Backend Infrastructure
- **Complete Remote API Implementation**: Full-featured backend APIs for remote workload deployment located in `scripts/Setup/remote/`
  - **Item CRUD API**: Create, Read, Update, Delete operations with support for soft delete and restore
  - **Endpoint Resolution API**: Dynamic endpoint resolution for Fabric services
  - **Authentication Service**: OAuth 2.0 token-based authentication with support for AAD login
  - **Token Exchange Service**: Secure token exchange for cross-service authentication
  - **OneLake Client Service**: Full OneLake integration for file operations and data access
  - **Permissions API**: Role-based access control and permission checking

### Job Scheduling Framework

#### Complete Job Lifecycle Management
- **Job Scheduler Controller**: Full TypeScript implementation for job scheduling operations
- **Job Instance API**: REST API endpoints for job execution management
- **Job Logging**: Comprehensive logging with OneLake integration for job execution history
- **Job State Management**: Track job status, progress, and completion states

#### Item Integration
- **Job Support**: Job scheduling integrated into the HelloWorldItem template
- **Context Menu Integration**: Run job options added to item context menus
- **Ribbon Actions**: Job execution controls in the item editor ribbon
- **Job Details Pane**: Full support for job details visualization

### Item Lifecycle Operations

#### Soft Delete and Restore
- **Soft Delete Endpoints**: Items can now be soft deleted instead of permanent deletion
- **Restore Functionality**: Restore soft-deleted items from the recycle bin
- **Lifecycle Operation Notifications**: Explicit lifecycle notifications for platform integration

### Variable Library Integration

#### New Client SDK Features
- **ApiVariableLibrary Component**: New playground component for Variable Library API testing

## 🔧 Technical Improvements

### Script Improvements

#### Setup and Configuration
- **SetupWorkload.ps1**: Major refactoring for remote hosting support
- **SetupConfiguration.md**: Updated documentation for new configuration options

#### Build Scripts
- **BuildManifestPackage.ps1**: Removed hosting type conditions for simplified builds

## 🐛 Bug Fixes

- Fixed saveItem function to ensure save button is only disabled on successful save

## 🔄 Migration Guide

### Enabling Remote Hosting

```powershell
# First, create an AAD application for Remote hosting type
.\CreateDevAADApp.ps1 -HostingType "Remote" -ApplicationName <name> -WorkloadName <name> -TenantId <tenant id>

# Run the SwitchToRemoteHosting script from the scripts/Setup directory
.\SwitchToRemoteHosting.ps1

# Follow the prompts to configure:
# - Azure AD application settings (use the app created above)
# - Remote backend URL
# - OneLake configuration
```

### Enabling Job Scheduling

```powershell
# Run the CreateJob script to add job support to your items
.\CreateJob.ps1

# This will:
# - Update HelloWorldItem configuration
# - Add job scheduler controller
# - Configure job-related UI components
```

## 📋 Upgrade Instructions

1. **Enable Remote Hosting** ⚠️ **New Feature**
   - Create an AAD application by running `.\CreateDevAADApp.ps1 -HostingType "Remote" -ApplicationName <name> -WorkloadName <name> -TenantId <tenant id>` from `scripts/Setup/`
   - Run `SwitchToRemoteHosting.ps1` from `scripts/Setup/`
   - Configure Azure AD application settings using the app created in the first step
   - Update `.env` file with remote hosting variables
   - Deploy backend to your hosting environment

2. **Enable Job Scheduling** ⭐ **Optional**
   - Run `CreateJob.ps1` from `scripts/Setup/`
   - Review generated job controller code
   - Configure job-related manifest entries
   - Test job execution in development environment

3. **Verify Lifecycle Operations** ⭐ **Recommended**
   - Verify lifecycle operation notifications
   - Update any custom lifecycle handling code
